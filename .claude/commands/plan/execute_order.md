다음과 같이 ORDER파일에 따라서 구현을 시작한다.

▼구체적인 순서

1. /docs/current_sprint/의 가장 최신 ORDER파일을 연다.(최신 날짜, NN　오더 넘버가 가장 큰 파일)
2. ORDER 파일의 내용을 바탕으로 구현을 시작한다. 사용자가 특별히 지시한 사항이 있다면 반드시 따른다. 
  - 기본적으로는 페이즈단위로 진행을 하고 페이즈 별로 테스트를 수행한다. 그러나 특히 특정 페이즈까지 진행하라는 지시가 있다면 그 페이즈까지만 진행, 전체 구현의 경우에는 전체 구현을 완료한다.
  - 특별한 지시가 없다면, 사용자 테스트가 필요한 시점까지 진행하고 사용자 테스트를 요청한다.
3. 사용자의 구현 지시 단계까지 구현을 완료한다.
   1. 각 Phase가 완료될 때마다 테스트를 수행한다. AI가 자동화 테스트를 수행할 수 있는 부분은 AI가 테스트를 수행하고, 사용자가 수동으로 테스트해야하는 부분이 있다면 사용자가 테스트를 수행한다.
   2. 각각의 단계에서 API 단위의 자동화 테스트는 AI가 꼭 진행하고 다음단계로 넘어간다.
4. 구현이 완료되면 작업 결과를 기록한다.
   1. 개발 과정에 발생한 이슈나 리스크가 있다면 리스크 및 롤백 섹션에 기록한다
   2. TODO섹션에 추가 작업 항목을 기록한다.
