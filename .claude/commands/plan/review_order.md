다음과 같이 작업 계획서를 작성한다.

▼구체적인 순서

1. 현재 열려있는 ORDER파일의 내용을 확인한다.열려있지 않다면 /docs/current_sprint/의 가장 최신 ORDER파일을 연다.(최신 날짜, NN　오더 넘버가 가장 큰 파일)
2. 내용을 확인할때,　/docs/의 ARCHITECTURE.md, FRONTEND_ROUTES.md, FRONTEND_STRUCTURE.md, BACKEND_API.md, BACKEND_STRUCTURE.md, DATABASE_SCHEMA_OVERVIEW.md　등의 문서를 참고하여 전체 시스템 구조를 이해한다.
   - 기존 구성의 폴더 역할을 정확히 지킨다.
   - 새로운 폴더 구성을 만들때는 기존의 구조를 최대한 활용하고, 불필요한 중복을 피한다, 또한 확장성을 고려하며, 꼭 만들어야할 때는 먼저 확인을 받아라.
   - 소스코드는 기능단위로 가능하면 분리시켜서, 재사용성을 높이고, 파일하나의 크기가 너무 커지지 않도록 한다.
2. ORDER파일의 내용을 파탕으로 작업 계획을 작성한다. 아래 내용을 신규로 작성한다. 원본은 하단으로 밀어내고, 너가 분석해서 작성한 새로운 내용을 상단에 작성한다.
   - 작업 개요 : 사용자의 요구사항을 AI가 분석해서 어떤 작업을 해야할지 이해한 내용을 기입함
   - 목표
   - 작업 계획 (Phase별)
   - 구현 체크리스트
   - 테스트 시나리오
   - 작업 결과 (완료 후 추가)
